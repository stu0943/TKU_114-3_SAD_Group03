## 學生任務

### 任務 1：案例拆解

以小組為單位，針對「社團活動報名與場地申請系統」完成下列分析：

1. 列出至少 5 個利害關係人（stakeholders）。
2. 列出至少 3 個目前流程中的痛點（pain points）。
3. 列出至少 5 種系統需要保存的資料（data）。
4. 列出至少 3 條業務規則（business rules）或限制。
5. 列出至少 2 個可能造成系統失敗的風險（risks）。
輸出格式：

```
小組名稱：Group 03

利害關係人（Stakeholders）：
1.學生(參加活動的人)
2.課外活動組
3.社團幹部
4.場地管理員、持有者
5.社團教練、指導老師

痛點（Pain Points）：
1.場地容易重複預約
2.報名名單整理耗費時間
3.歷史紀錄難以查詢

資料（Data）：
1.使用者資料
2.場地資料
3.社團資料
4.活動資料
5.場地申請紀錄

業務規則（Business Rules）：
1.活動開始後不得再接受報名。
2.同一場地同一時段只能核准一筆借用申請。
3.場地申請須經課外活動組及場地管理員審核後才算完成。

風險（Risks）：
1.權限管理錯誤
2.資料遺失
```

完成標準：

- 每一欄都要能對應案例，不可只寫空泛詞。
- 利害關係人（stakeholder）不能只寫「使用者」，要具體。
- 痛點（pain point）需要描述目前問題，不是直接寫解法。

### 任務 2：候選題目發想

每組提出 2-3 個候選題目。每個題目需包含：

1. 系統名稱。
2. 目標使用者。
3. 要解決的問題。
4. 三個核心任務。
5. 系統不處理的範圍。

輸出格式：

```
候選題目 1：
系統名稱：演唱會同行媒合系統
目標使用者：演唱會觀眾／粉絲
要解決的問題：許多粉絲想參加演唱會，但因身邊沒有同行朋友或是追同團的人、座位相近卻互不認識，或希望與有相同興趣的人一起參與活動，導致參與意願降低。此系統提供安全且方便的同行媒合平台，協助使用者找到適合的同行夥伴。
核心任務：
1.建立同行需求（選擇演唱會、團體、場次、座位區域、個人偏好）
2.搜尋並配對符合條件的同行者
3.發送同行邀請並管理媒合結果（接受／拒絕／取消）
本期不處理：
1.不處理金流與票券交易。
2.不提供演唱會購票功能。
3.不提供即時聊天或社群討論功能（僅提供媒合結果通知）。
```
```
候選題目 2：
系統名稱：校園二手書交易平台
目標使用者：在校學生買家、賣家
要解決的問題：學生每學期購買大量教材，使用後閒置浪費；有需求的學生則難以快速找到價格合理的二手書。本系統提供校園內二手書刊登與交易資訊管理，提升教材再利用率。
核心任務：
1.刊登二手書資訊（書名、課程、價格、書況）
2.搜尋並瀏覽符合需求的二手書
3.管理交易狀態（預約、已售出、取消刊登）
本期不處理：
1.不提供線上付款功能。
2.不負責物流配送。
3.不處理校外人士的交易。
```


完成標準：

- 至少 2 個候選題目。
- 每個題目都要能在 5 週內做出原型（prototype）。
- 題目不能只描述技術，例如「人工智慧系統（AI system）」，必須描述使用情境。

### 任務 3：候選題目評選

每組從候選題目中選出 1 個主題。使用下列表格評估：

| 評估項目                        | 題目 1 | 題目 2 |
| ------------------------------- | ------ | ------ |
| 使用者是否明確                  |    v    |    v    |
| 是否有 3 個以上核心任務         |    v    |    v    |
| 是否有資料需要管理              |    v    |    v    |
| 是否有角色、權限或規則          |    v    |    v    |
| 5 週內是否能做原型（prototype） |    v    |    v    |
| 是否容易訪談或模擬需求          |    x    |    v    |

完成標準：

- 至少比較 2 個候選題目。
- 最後選出 1 個主題，並寫出選擇理由。

### 任務 4：撰寫問題陳述（problem statement）

使用句型完成問題陳述（problem statement）：

```
對於 [目標使用者（target users）] 而言，目前在 [目前情境（current situation）] 中，常發生 [痛點與影響（pain / impact）]。
本專案希望設計一個 [系統概念（system concept）]，協助 [期待改善（desired change）]，並在本期聚焦於 [範圍焦點（scope focus）]。
```

完成標準：

- 至少 120 字。
- 必須包含目標使用者（target users）、目前情境（current situation）、痛點與影響（pain / impact）、期待改善（desired change）、範圍焦點（scope focus）。
- 不可以只列功能。

### 任務 5：建立範圍陳述（scope statement）

每組完成：

1. 專案目標（project goal）：1-2 句。
2. 範圍內項目（in-scope）：至少 5 項。
3. 範圍外項目（out-of-scope）：至少 5 項。
4. 核心任務（core tasks）：至少 3 個可展示任務。
5. 初步成功標準（initial success criteria）：至少 3 項。
