## 學生任務

### 任務 1：案例拆解

以小組為單位，針對「社團活動報名與場地申請系統」完成下列分析：

1. 列出至少 5 個利害關係人（stakeholders）。
2. 列出至少 3 個目前流程中的痛點（pain points）。
3. 列出至少 5 種系統需要保存的資料（data）。
4. 列出至少 3 條業務規則（business rules）或限制。
5. 列出至少 2 個可能造成系統失敗的風險（risks）。
輸出格式：

```
小組名稱：

利害關係人（Stakeholders）：
1.
2.
3.
4.
5.

痛點（Pain Points）：
1.
2.
3.

資料（Data）：
1.
2.
3.
4.
5.

業務規則（Business Rules）：
1.
2.
3.

風險（Risks）：
1.
2.
```

完成標準：

- 每一欄都要能對應案例，不可只寫空泛詞。
- 利害關係人（stakeholder）不能只寫「使用者」，要具體。
- 痛點（pain point）需要描述目前問題，不是直接寫解法。

### 任務 2：候選題目發想

每組提出 2-3 個候選題目。每個題目需包含：

1. 系統名稱。
2. 目標使用者。
3. 要解決的問題。
4. 三個核心任務。
5. 系統不處理的範圍。

輸出格式：

```
候選題目 1：
系統名稱：
目標使用者：
要解決的問題：
核心任務：
1.
2.
3.
本期不處理：
```

完成標準：

- 至少 2 個候選題目。
- 每個題目都要能在 5 週內做出原型（prototype）。
- 題目不能只描述技術，例如「人工智慧系統（AI system）」，必須描述使用情境。

### 任務 3：候選題目評選

每組從候選題目中選出 1 個主題。使用下列表格評估：

| 評估項目                        | 題目 A | 題目 B | 題目 C |
| ------------------------------- | ------ | ------ | ------ |
| 使用者是否明確                  |        |        |        |
| 是否有 3 個以上核心任務         |        |        |        |
| 是否有資料需要管理              |        |        |        |
| 是否有角色、權限或規則          |        |        |        |
| 5 週內是否能做原型（prototype） |        |        |        |
| 是否容易訪談或模擬需求          |        |        |        |

完成標準：

- 至少比較 2 個候選題目。
- 最後選出 1 個主題，並寫出選擇理由。

### 任務 4：撰寫問題陳述（problem statement）

使用句型完成問題陳述（problem statement）：

```
對於 [目標使用者（target users）] 而言，目前在 [目前情境（current situation）] 中，常發生 [痛點與影響（pain / impact）]。
本專案希望設計一個 [系統概念（system concept）]，協助 [期待改善（desired change）]，並在本期聚焦於 [範圍焦點（scope focus）]。
```

完成標準：

- 至少 120 字。
- 必須包含目標使用者（target users）、目前情境（current situation）、痛點與影響（pain / impact）、期待改善（desired change）、範圍焦點（scope focus）。
- 不可以只列功能。

### 任務 5：建立範圍陳述（scope statement）

每組完成：

1. 專案目標（project goal）：1-2 句。
2. 範圍內項目（in-scope）：至少 5 項。
3. 範圍外項目（out-of-scope）：至少 5 項。
4. 核心任務（core tasks）：至少 3 個可展示任務。
5. 初步成功標準（initial success criteria）：至少 3 項。

## 生成式 AI 使用規範與紀錄（Generative AI Use Log）

本課可以使用生成式 AI（generative AI）協助發想、整理、診斷與檢查，但不能把生成式 AI（generative AI）產出直接當成小組最終成果。

每次使用生成式 AI（generative AI）後，請記錄：

1. 使用目的。
2. 提示詞（prompt）摘要。
3. 生成式 AI（generative AI）給出的重點。
4. 小組採用哪些建議。
5. 小組不採用哪些建議。
6. 人工修正內容。
